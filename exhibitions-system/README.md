# نظام إدارة المعارض والمخزون

نظام مستقل لإدارة شغل تاجر: بضاعة من موردين → مستودعات → معارض مؤقتة → بيع عبر موظفين متنقّلين، مع عُهدة (بضاعة + كاش)، رواتب يومية + سُلف + عمولات، ومحاسبة كل معرض كمركز ربح مستقل.

تقنيًا = **POS + إدارة مخزون متعدّد المواقع + عُهدة + رواتب/HR + محاسبة مراكز ربح**.

## الأدوار (3)

- **الأدمن (المالك):** كل شي — منتجات/تكلفة، معارض/تارجت/عمولة، موظفين/رواتب، تأكيد العهدة والدفعات، كل التقارير والأرباح.
- **مدير المخزون / أمين المستودع:** الكميات، الموافقة على الطلبات، التحويلات، استلام مرتجعات الإغلاق، الجملة. **ما يشوف الأرباح/التكلفة.**
- **الموظف:** دخول بـ (جوال + كود). يبيع (يحدّد السعر)، يطلب بضاعة، يسجّل سحوبات، يسلّم العهدة. يشوف: مبيعاته، عهدته، التارجت، راتبه. **ما يشوف التكلفة أبدًا.**

## ⚠️ القرار المفتوح الوحيد: مين بياخد العمولة؟

التارجت والعمولة على مستوى المعرض، والموظفين بيتنقّلوا بحرية. الجداول جاهزة لكل الخيارات:
- `branches.commission_mode` enum: `single_manager` | `proportional` | `manual_pool` (حاليًا NULL لحين الحسم).
- `branches.manager_id` لخيار "مسؤول معرض واحد".
- `commissions.beneficiary_id` (nullable) لتحديد المستفيد فعليًا.

**الخيارات:** (أ) مسؤول معرض واحد ياخد الكل — (ب) توزيع نسبي حسب مبيعات كل موظف — (ج) تجميع كمبلغ والأدمن يوزّع يدويًا.
