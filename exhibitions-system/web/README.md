# واجهة نظام إدارة المعارض والمخزون

واجهة أمامية عربية (RTL) مبنية بـ **Vite + React + TypeScript + Tailwind CSS** متصلة بـ **Supabase**.

---

## ⚠️ خطوة إلزامية قبل أي شيء

> **يجب تفعيل سكيمة `exhibitions` في Supabase:**
> **Settings → API → Exposed schemas → أضف `exhibitions`**
>
> كل جداول ودوال هذا النظام تعيش في سكيمة `exhibitions` (وليست `public`). **لن تتمكّن الواجهة من التواصل مع قاعدة البيانات حتى تُضاف هذه السكيمة لقائمة Exposed schemas.**

---

## التشغيل

```bash
npm install
npm run dev
```

ثم افتح الرابط الظاهر في الطرفية (عادة `http://localhost:5173`).

للبناء للإنتاج:

```bash
npm run build
npm run preview
```

موجودة في ملف `.env` (مُضمّن في المستودع لأن المفتاح المنشور عام بطبيعته):

```
VITE_SUPABASE_URL=https://axzqbqzdvtlbgbwzeiry.supabase.co
VITE_SUPABASE_ANON_KEY=sb_publishable_9HzBp1FRow_JM8a5Hs81Ag_OZ48Y3Xi
```

> الكود يقرأ هذه القيم من `import.meta.env` مع قيمة احتياطية ثابتة في `src/lib/supabase.ts`،
> لذلك يعمل التطبيق حتى بدون ملف `.env`. انسخ `.env.example` إلى `.env` عند الحاجة لتغيير المشروع.

عميل Supabase يُنشأ مع `db: { schema: 'exhibitions' }`.

## الأدوار الثلاثة وكيفية الدخول

من الصفحة الرئيسية يوجد مدخلان: **دخول الإدارة** و **دخول الموظف**.

### 1) الأدمن (Admin)
- يدخل عبر **Supabase Auth** (بريد + كلمة مرور) من `/admin/login`.
- **أول مستخدم يُسجَّل على الإطلاق يصبح أدمن تلقائيًا.**
- بعد الدخول: يُستدعى `ensure_my_profile` ثم `my_profile` لتحديد الدور.
- يرى كل شيء: المنتجات (مع التكلفة)، الكتالوج، المعارض، الموظفين، الطلبات، المالية، الجملة، المخزون، استلام البضاعة، أرباح المعارض والعمولات.

### 2) مدير المخزون (Inventory Manager)
- يدخل عبر **Supabase Auth** أيضًا من `/admin/login` (التسجيلات اللاحقة بعد الأدمن الأول تصبح مدير مخزون).
- يفعّل الأدمن صلاحياته لاحقًا (استلام بضاعة / مراجعة الطلبات / التحويلات / الجملة / المرتجعات).
- **لا يرى التكلفة أو الأرباح إطلاقًا** — الشاشات تستخدم `products_public` فقط.

### 3) الموظف (Employee)
- **لا يستخدم Supabase Auth.** يدخل بـ **رقم الجوال + كود الوصول** من `/employee/login`.
- يُستدعى `employee_login` ويُخزَّن الـ token في `localStorage`، ويُمرَّر في كل العمليات.
- شاشاته: لوحة التحكم (مبيعات اليوم + التقدّم نحو الهدف + العُهدة)، نقطة البيع، طلب بضاعة، سحب عُهدة، تسليم العُهدة.

## البنية

```
web/
├─ src/
│  ├─ lib/
│  │  ├─ supabase.ts        عميل Supabase (سكيمة exhibitions)
│  │  ├─ api.ts             غلاف موحّد لكل دوال RPC مع معالجة الأخطاء
│  │  ├─ types.ts           الأنواع
│  │  └─ format.ts          تنسيق العملة "ر.س"
│  ├─ context/              سياقات الدخول (أدمن/موظف) واختيار المعرض
│  ├─ components/           مكوّنات واجهة مشتركة + منتقي بنود المنتجات
│  ├─ pages/
│  │  ├─ Landing.tsx
│  │  ├─ employee/          شاشات الموظف
│  │  └─ admin/             شاشات الأدمن / مدير المخزون
│  ├─ App.tsx               المسارات
│  └─ main.tsx
└─ ...ملفات الإعداد
```

كل استدعاءات RPC تمرّ عبر `src/lib/api.ts` التي ترمي رسالة الخطأ العربية القادمة من قاعدة البيانات (`error.message`).
