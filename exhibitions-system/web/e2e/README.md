# اختبارات الواجهة (E2E) — Playwright

اختبارات تفتح الموقع كمستخدم حقيقي وتفحص المسارات تلقائيًا. **شغّلها من جهازك** (الموقع مفتوح عندك)، لأن بيئة التطوير السحابية تحجب الموقع.

## التشغيل (من مجلد `exhibitions-system/web`)

```bash
npm install
npx playwright install chromium      # أول مرة فقط — يثبّت المتصفّح
cp e2e/.env.example e2e/.env         # ثم املأ القيم
npm run e2e                          # تشغيل كل الاختبارات
npm run e2e:ui                       # واجهة تفاعلية لمشاهدة الخطوات
npm run e2e:report                   # عرض تقرير آخر تشغيل (HTML + لقطات الفشل)
```

## الإعداد (`e2e/.env`)

| المتغيّر | الوصف |
|---|---|
| `E2E_BASE_URL` | رابط الموقع — `https://www.sindbadsa.com` أو `http://localhost:5173` |
| `E2E_OWNER_EMAIL` / `E2E_OWNER_PASSWORD` | إيميل وكلمة مرور حساب مالك مطعم (حسابك) |
| `E2E_EMP_PHONE` / `E2E_EMP_CODE` | بيانات موظف (جاهزة لقهوة الساعة: `0999000001` / `472913`) |
| `E2E_TENANT_ID` | معرّف المستأجر لرابط `/menu/:id` (قهوة الساعة جاهز) |
| `E2E_RUN_FLOWS` | `1` لتشغيل اختبار يُنشئ بيانات فعلية (فتح طاولة/فاتورة). الأفضل على مستأجر تجريبي |

> الاختبارات التي تنقصها بياناتها تُتخطّى تلقائيًا (skip) ولا تفشل.

## ما الذي يُفحَص

- **public.spec** — صفحات الدخول (مشترك/موظف)، فتح الجذر، تحميل منيو الطلب أونلاين `/menu` بلا انهيار أو أخطاء كونسول.
- **owner.spec** — دخول المالك، لوحة تحكم المطعم تعرض «مبيعات اليوم»، **عدم خلط أقسام التجزئة** (لا «المعارض»/«الجملة»)، فتح التحليلات وشاشة الطاولات.
- **employee.spec** — دخول الموظف، فتح شاشتي الطاولات والمطبخ.
- **restaurant-flow.spec** (اختياري) — تدفّق كامل: فتح طاولة → صنف → إرسال للمطبخ → إقفال ودفع.

## ملاحظات
- الاختبارات تلتقط **أخطاء الكونسول (JS)** وتُفشل عند وجودها في الصفحات الحرجة — هذا يكشف الأعطال الصامتة.
- عند الفشل تُحفظ **لقطة شاشة + فيديو + trace** في `playwright-report/` و`test-results/`.
- إن تغيّرت نصوص الأزرار في الواجهة، حدّث المُحدِّدات (selectors) في `e2e/tests/`.
